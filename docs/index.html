Welcome to the C Utils Project Official Documentation Site

Summary

The C Utils Project, a.k.a. cutils, is a set of modular, lightweight libraries providing basic, common utilities for C/C++ projects. I am a student and these libraries exist primarily for learning and personal use, not geared towards production or professional use cases. In the name of good practice (and for the small chance someone sees this in the future!) I am still putting effort into its documentation.

As an extra point, this project serves to replace and improve upon an older iteration, zetaml, which was more specifically a maths library for graphics development.

The C Utils Project, cutils, is Free and Open Source Software, and the source repository can be found here. I am not affiliated with the C Standard, any compilers, or other projects of a similar name.

Libraries

As mentioned earlier, the C Utils Project is not actually a library itself, but rather a collection of smaller libraries that can be used independent of each other. These libraries are limited to the key features they aim to provide, and so they are lightweight and normally simple to use. Some libraries are bigger than others!

  • clist: Linked list implementation

    The clist library provides a basic doubly linked list structure, as well as a multitude of functions for using lists.

  • cio: Console input/output and logging functions

    The cio library provides extra cross-platform utility functions for console output and log messages.

  • carray: Dynamic array implementation

    The carray library provides an implementation of a dynamic array structure with a variety of extra operations.

Compiling C Utility Libraries

Libraries within the C Utils collection use CMake for building/compiling.

The command below compiles all utility libraries in release configuration:
cmake . -B build/ -DCMAKE_BUILD_TYPE=Release

Generally speaking, you do not need to compile the libraries in debug configuration unless you are contributing (tysm ^^) or otherwise just want to compile test executables alongside them. On that note, to compile tests (using the GoogleTest framework), build with debug configuration and the -DCUTILS_TESTS=ON flag. Note that -Wall + equivalents are enabled when in debug mode, so all warnings are shown and they are treated as errors.

Individual utility libraries can be excluded from the build chain with individual compilation flags. They correspond to the relevant library name; see the root CMakeLists.txt for specific option names. For example, -DCUTILS_CLIST=OFF disables compiling the clist library. All libraries are compiled by default.

Compiling HTML documentation

The documentation you're looking at right now is also compiled using CMake. It can be accessed through its deployment via GitHub Pages, or compiled locally as part of the build pipeline. This manual does not use any automation tools like documentation generators (e.g. Doxygen), just simple hand-typed HTML and CSS, as you can probably tell. However, some parts (like the header) are templated for convenience, hence the need for CMake.

To build the documentation yourself, apply the -DCUTILS_DOCS=ON flag. This is not enabled by default!
